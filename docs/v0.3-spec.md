# DaxLib.SVG v0.3 Specification

## Status
- Draft for implementation planning
- Target branch: `0.3.0` (current branch) (forward-compatible with `main`)

## Objectives
- Reduce repeated table-shaping logic currently duplicated as `_Data` in many `Compound.*` functions.
- Add standardized orientation support (`Vertical`/`Horizontal`) where meaningful.
- Add standardized axis rendering toggles for chart compounds.
- Introduce `Visual.*` wrappers with Fluent 2–style prebaked defaults and minimal parameter surfaces.
- Fill gaps in chart coverage, including a new **multi-pill wrapping** compound.

## Non-Goals
- Breaking changes to existing `DaxLib.SVG.Compound.*` signatures in v0.3.
- A full Fluent 2 design-token system in this release.
- Adding model-specific assumptions tied to a single semantic model.

## 1) Shared Data UDFs

### 1.1 `DaxLib.SVG.Data.AxisMeasure`
Shared helper that produces the standard data table previously duplicated in compound functions.

```dax
function 'DaxLib.SVG.Data.AxisMeasure' =
(
    axisRefColumn: ANYREF EXPR,
    measureRefExpr: NUMERIC EXPR,
    axisMode: STRING,
    includeBlanks: BOOLEAN
) => TABLE
```

#### Behavior
- Returns table columns:
  - `@AxisValue` (raw axis value)
  - `@AxisIndex` (normalized axis index used for plotting and ordering)
  - `@Value` (measure result)
- `axisMode` values:
  - `"Auto"` (default): infer numeric/date axis vs categorical axis
  - `"Numeric"`: force numeric/date semantics
  - `"Categorical"`: force dense ranking semantics
- `includeBlanks`:
  - `FALSE` (default): rows with blank `@Value` are removed
  - `TRUE`: retain rows with blank `@Value`

#### Notes
- Must avoid `SUMMARIZECOLUMNS()` for cross-mode compatibility.
- Must avoid table-returning `IF()` branching patterns.
- Use deterministic ordering by `@AxisIndex` then `@AxisValue`.

### 1.2 `DaxLib.SVG.Data.Range`
Utility for common min/max/baseline calculations.

```dax
function 'DaxLib.SVG.Data.Range' =
(
    dataTable: TABLE,
    valueColumn: ANYREF EXPR,
    includeZero: BOOLEAN
) => TABLE
```

#### Returned columns
- `@Min`
- `@Max`
- `@Baseline` (`0` when `includeZero=TRUE` and range is fully positive; else `@Min`)

## 2) Standard Compound Behavior Contracts

## 2.1 Orientation
For orientation-capable compounds, add:

```dax
orientation: STRING
```

Allowed values:
- `"Vertical"` (default)
- `"Horizontal"`

Applicability in v0.3:
- Required: `Compound.Bar`, `Compound.Line`, `Compound.Area`, `Compound.Boxplot`, `Compound.Pills`
- Deferred: `Compound.Violin`, `Compound.Heatmap` (future matrix orientation support)

## 2.2 Axis integration (inside each compound)
For axis-capable compounds, axis rendering is part of the compound itself (not a separate public axis UDF).

```dax
showAxis: BOOLEAN,
axisFontSize: INT64
```

Defaults:
- `showAxis = FALSE`
- `axisFontSize = 10`

Rendering rules:
- Axis rendering is part of compound layout and MUST adjust plotting bounds before mark scaling.
- When `showAxis=TRUE`, compounds reserve axis space using font-size-aware paddings.
- Axis style should follow Power BI-style visuals:
  - Font family: `Segoe UI`
  - Tick label color: neutral gray
  - Axis line: 1px subtle gray

Layout accounting rules:
- X-axis reserved height = `axisFontSize + tickPadding + axisLinePadding`.
- Y-axis reserved width = `maxTickLabelWidth + tickPadding + axisLinePadding`.
- `maxTickLabelWidth` should be estimated from formatted tick strings and Segoe UI width heuristic.
- Plot area (`plotX`, `plotY`, `plotWidth`, `plotHeight`) is derived after subtracting reserved axis space.
- All value normalization for bars/lines/areas/boxplots must use plot bounds (not raw container bounds).

---

## 3) Axis Rendering Model (non-public)

Axis generation is implemented as internal variable blocks inside each axis-capable compound.

- No public `DaxLib.SVG.Axis.*` function surface in v0.3.
- Internal axis block responsibilities:
  - Tick generation
  - Tick label formatting
  - Font-size-aware reserved-space calculation
  - Axis SVG generation in Power BI style
- This preserves correctness because scaling and axis layout are computed together in one function context.

## 4) New `Visual.*` Layer (Fluent 2–style defaults)

`Visual.*` functions are wrappers over `Compound.*` with minimal parameters, prebaked style, and predictable defaults.

## 4.1 Naming
- `DaxLib.SVG.Visual.Bar`
- `DaxLib.SVG.Visual.Line`
- `DaxLib.SVG.Visual.Area`
- `DaxLib.SVG.Visual.Pills`

## 4.2 Example signatures

```dax
function 'DaxLib.SVG.Visual.Bar' =
(
    axisRefColumn: ANYREF EXPR,
    measureRefExpr: NUMERIC EXPR,
    color: STRING,
    orientation: STRING,
    showAxis: BOOLEAN
) => STRING
```

```dax
function 'DaxLib.SVG.Visual.Line' =
(
    axisRefColumn: ANYREF EXPR,
    measureRefExpr: NUMERIC EXPR,
    color: STRING,
    orientation: STRING,
    showAxis: BOOLEAN
) => STRING
```

```dax
function 'DaxLib.SVG.Visual.Pills' =
(
    axisRefColumn: ANYREF EXPR,
    measureRefExpr: NUMERIC EXPR,
    color: STRING,
    maxRows: INT64
) => STRING
```

### 4.3 Fluent 2 defaults (prebaked)
- Font family: `Segoe UI`
- Shape corner radius: medium/rounded
- Subtle fill opacity and clear 1px stroke
- Balanced internal spacing and text centering
- Axis (when enabled): Power BI-style subtle gray axis/ticks with Segoe UI labels
- Optional color fallback chain:
  - explicit input color
  - theme fallback (`DaxLib.SVG.Color.Theme`)
  - hard fallback `#0F6CBD`

## 5) New Compound: Wrapping Multi-Pill Layout

## 5.1 Function

```dax
function 'DaxLib.SVG.Compound.Pills' =
(
    x: INT64,
    y: INT64,
    width: INT64,
    height: INT64,
    paddingX: DOUBLE,
    paddingY: DOUBLE,
    axisRefColumn: ANYREF EXPR,
    labelExpr: STRING EXPR,
    valueExpr: NUMERIC EXPR,
    colorExpr: STRING EXPR,
    pillHeight: INT64,
    minPillWidth: INT64,
    maxPillWidth: INT64,
    gapX: INT64,
    gapY: INT64,
    maxRows: INT64,
    overflowMode: STRING,
    sortMode: STRING
) => STRING
```

## 5.2 Purpose
Renders multiple pills in a bounded canvas and wraps to new lines when the next pill would exceed available row width.

## 5.3 Layout model
- Establish content bounds from `(x, y, width, height)` and paddings.
- Compute each pill width using text-length heuristic + optional value influence:
  - `pillWidth = clamp(estimatedLabelWidth + internalPadding, minPillWidth, maxPillWidth)`
- Place pills left-to-right until `nextX + pillWidth > contentRight`.
- When exceeded, wrap:
  - `currentRow += 1`
  - `currentX = contentLeft`
  - `currentY += pillHeight + gapY`
- Stop when row limit exceeded or next row exceeds content height.

## 5.4 Sorting
`sortMode` values:
- `"AxisAsc"` (default)
- `"AxisDesc"`
- `"ValueDesc"`
- `"ValueAsc"`

## 5.5 Overflow handling
`overflowMode` values:
- `"Clip"` (default): silently stop rendering overflow pills
- `"EllipsisPill"`: reserve last visible slot for a final `"+N"` summary pill
- `"ScaleDown"` (deferred): reduce width heuristics to fit more pills before wrapping

## 5.6 Color behavior
- `colorExpr` can be per-row color.
- If row color blank, fallback to neutral fluent token color.

## 5.7 Output
- Concatenated SVG `<rect>` + `<text>` elements (or composed via existing `Compound.Pill` internals).
- Returns blank if no data rows.

## 5.8 Performance constraints
- Data preparation must use shared `Data.AxisMeasure`.
- Avoid repeating expensive expression evaluation inside layout loops.
- Keep per-row string output compact.

## 5.9 Acceptance criteria
- Pills wrap consistently for any width and deterministic sorting.
- No overlap across rows.
- Clip/ellipsis behavior is deterministic.
- Works with both numeric and categorical axis refs.

## 11) Implementation Appendix: Axis Constants

These constants are normative defaults for v0.3 and should be used across axis-capable compounds.

### 11.1 Constants
- `axisFontFamily = "Segoe UI"`
- `axisFontSize = 8` (overridable via parameter)
- `axisLineStrokeWidth = 1`
- `tickLength = 3`
- `tickPadding = 4`
- `axisLinePadding = 2`
- `axisColor = "#605E5C"`
- `labelWidthFactor = 0.56` (Segoe UI average glyph-width factor)

### 11.2 Label width estimate
- `estimatedLabelWidth = LEN( formattedLabel ) * axisFontSize * labelWidthFactor`
- `maxTickLabelWidth = MAXX( tickTable, estimatedLabelWidth )`

### 11.3 Reserved space formulas
- `reservedXAxisHeight = axisFontSize + tickLength + tickPadding + axisLinePadding`
- `reservedYAxisWidth = maxTickLabelWidth + tickLength + tickPadding + axisLinePadding`

### 11.4 Plot-area derivation
- `plotX = x + IF( showAxis, reservedYAxisWidth, 0 )`
- `plotY = y`
- `plotWidth = width - IF( showAxis, reservedYAxisWidth, 0 )`
- `plotHeight = height - IF( showAxis, reservedXAxisHeight, 0 )`

If `showAxis = FALSE`, reserved values are zero and plot area equals the original drawing area.

### 11.5 Tick defaults
- `tickCountX = 5`
- `tickCountY = 4`
- Tick labels should be preformatted once per axis and reused for both width estimation and rendering.

### 11.6 Bounds safety
- Clamp all reserved values to prevent negative plot dimensions.
- If `plotWidth <= 0` or `plotHeight <= 0`, return blank output.
- Axis/group strings should be emitted after marks when layering requires labels to remain visible.
